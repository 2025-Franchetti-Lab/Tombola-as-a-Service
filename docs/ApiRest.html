Convenzioni generali

AspettoValore
Base URLhttps://<host>/api
Formato datiJSON — Content-Type: application/json
AutenticazioneBearer token JWT nell'header Authorization: Bearer <token>

1. Utenti — /api/utente

POST /api/utente/registra

Registra un nuovo utente nella piattaforma.

Request body
{
  "nome":       "Mario",
  "cognome":    "Rossi",
  "email":      "mario.rossi@example.com",
  "authMethod": "password",
  "password":   "s3cr3t!"
}
201 Created
{
  "utenteId":   1,
  "nome":       "Mario",
  "cognome":    "Rossi",
  "email":      "mario.rossi@example.com",
  "authMethod": "password",
  "createdAt":  "2025-12-01T10:00:00Z"
}
POST /api/utente/login

Autentica l'utente e restituisce un token JWT.

Request body
{
  "email":      "mario.rossi@example.com",
  "authMethod": "password",
  "credential": "s3cr3t!"
}
200 OK
{
  "token":    "<JWT>",
  "scadeAt":  "2025-12-01T22:00:00Z",
  "utenteId": 1
}
GET /api/utente/lista Gestore

Elenco degli utenti registrati, con filtro.

Query params
ParametroTipoDefaultDescrizione
cercastring—Filtro su nome, cognome o email
200 OK
{
  "totale": 42,
  "utenti": [
    { "utenteId": 1, "nome": "Mario", "cognome": "Rossi", "email": "...", "attivo": true }
  ]
}
GET /api/utente/{utenteId}

Restituisce i dettagli di un singolo utente.

200 OK
{
  "utenteId":  1,
  "nome":      "Mario",
  "cognome":   "Rossi",
  "email":     "mario.rossi@example.com",
  "authMethod":"password",
  "attivo":    true,
  "createdAt": "2025-12-01T10:00:00Z"
}
PUT /api/utente/{utenteId}

Aggiorna i dati di un utente. L'utente può modificare solo i propri dati; il gestore può modificare qualsiasi utente.

Request body (tutti i campi opzionali)
{
  "nome":     "Mario",
  "cognome":  "Bianchi",
  "email":    "mario.bianchi@example.com",
  "password": "nuovaSicura99!"
}
200 OK — oggetto utente aggiornato
DELETE /api/utente/{utenteId} Gestore

Disabilita (soft-delete) un utente. Il record rimane nel database con attivo = 0.

200 OK
{ "messaggio": "Utente disabilitato correttamente" }

2. Tombolate — /api/tombolata

Crea una nuova tombolata. Lo stato iniziale è creata.

  "dataFineAssegnazioneCartelle": "2025-12-25T20:30:00Z"

  "nome":                         "Tombola di Natale 2025",
  "gestoreNome":                  "Luigi Verdi",

  "dataAttivazione":              "2025-12-25T21:00:00Z",
  "dataFineAssegnazioneCartelle": "2025-12-25T20:30:00Z",
  "stato":                        "creata",
  "createdAt":                    "2025-12-01T09:00:00Z"

PUT /api/tombolata/{tombolataId}/attiva Gestore

Avvia il gioco (stato → attiva). Da usare dopo il termine della deadline assegnazione cartelle.

PUT /api/tombolata/{tombolataId}/termina Gestore

Chiude la tombolata (stato → terminata). Nessuna scrittura ulteriore sarà possibile.

200 OK

3. Cartelle — /api/cartella

POST /api/cartella/crea Gestore

Crea una o più cartelle per una tombolata. La generazione può essere automatica o manuale.

Request body — generazione automatica
{ "tombolataId": 1, "quantita": 3 }
Request body — generazione manuale (15 numeri con posizione)
{
  "tombolataId": 1,
  "numeri": [
    { "numero": 5,  "riga": 1, "colonna": 1 },
    { "numero": 17, "riga": 1, "colonna": 2 },
    // ... 13 altri numeri
  ]
}
I 15 numeri devono rispettare la struttura: 3 righe, 9 colonne, 5 numeri per riga. La colonna deve corrispondere al decile del numero (colonna 2 → numeri 10–19, ecc.).
201 Created
{
  "cartelle": [
    { "cartellaId": 10, "tombolataId": 1, "utenteId": null },
    { "cartellaId": 11, "tombolataId": 1, "utenteId": null }
  ]
}
422 — numeri duplicati, colonna incongruente con decile, ecc.
POST /api/cartella/assegna Gestore

Assegna una cartella esistente a un utente. Verifica che nessun numero della cartella si ripeta tra le cartelle già assegnate all'utente nella stessa tombolata.

Request body
{ "cartellaId": 10, "utenteId": 5 }
200 OK
{ "cartellaId": 10, "utenteId": 5, "updatedAt": "2025-12-10T09:00:00Z" }
422 — numero già presente in un'altra cartella dell'utente nella stessa tombolata
GET /api/cartella/{cartellaId}

Dettaglio della cartella con stato di ogni numero (estratto / non estratto).

200 OK
{
  "cartellaId":  10,
  "tombolataId": 1,
  "utenteId":    5,
  "numeri": [
    { "numero": 5,  "riga": 1, "colonna": 1, "estratto": true  },
    { "numero": 17, "riga": 1, "colonna": 2, "estratto": false }
  ]
}

Stato sintetico della cartella, aggregato per riga.

200 OK
{

  "righe": [
    {
      "riga":           1,
      "numeriEstratti": 2,
      "dettaglio": [
        { "numero": 5,  "estratto": true  },
        { "numero": 17, "estratto": false },
        { "numero": 23, "estratto": true  },
        { "numero": 44, "estratto": false },
        { "numero": 78, "estratto": false }
      ]
    },
    { "riga": 2, "numeriEstratti": 1, "dettaglio": [...] },
    { "riga": 3, "numeriEstratti": 0, "dettaglio": [...] }
  ],
  "totaleEstratti":  3,
  "vinciteRaggiunte": ["ambo"]
}

Elimina una cartella. Consentito solo se la tombolata è in stato creata o aperta.

Regole di business

  1. Unicità numeri per utente — un utente non può avere lo stesso numero in due cartelle diverse della stessa tombolata.
  2. Struttura cartella — 3 righe, 9 colonne, 5 numeri per riga, 15 totali. Colonna determinata dal decile del numero.
  3. Coerenza colonna/decile — verificata a livello DB con vincolo CHECK.
  4. Vincite in ordine crescente — ambo → terno → quaterna → cinquina → tombola.
  5. Esclusione vincitore — un utente che ha raggiunto il livello N non concorre al livello N+1.
  6. Estrazione senza ripetizione — ogni numero può essere estratto al massimo una volta per tombolata.
  7. Deadline assegnazione — dopo dataFineAssegnazioneCartelle non è possibile assegnare nuove cartelle.
  8. Transizione stati unidirezionale — creata → aperta → attiva → terminata.
